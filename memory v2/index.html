Text Defense

  • High Concept: Test your memory skills by correctly typing the sequences shown to you.
  • Theme: I did not use any of the optional themes provided.
  • Mandated Variety: Keyboard input. More random. Play style is other: memory game
  • Player Experience Goals: I want players to feel pressed for time, trying to memorize as many characters as they can.
  • Gameplay: The core mechanic is memorization. Players try to memorize as many characters in the sequence as possible, and then try to recreate it in the text field. The game is simple, but gets more difficult the longer you play it. The sequences get longer, and the amount of time you get to analyze it gets shorter.
  • Prototype Goal: This game is prototyping a basic memory mechanic. Does the mechanic work as intended, lending itself towards a overall enjoyable and fun gaming experience?
  • Features: Something very unique about this game is the text input. It was very difficult to get it to work, and I don't think games from other students feature such a mechanic.
  • Setting/Premise: The game does not have any characters.
  • Story: The game does not have a story surrounding it.
  • Target Audience: The game could be played by anyone, but will most likely be enjoyed most by people who are fond of memory games.
  • Play Time: The game does not have a set end point, but it takes about 3-5 minutes to reach the most difficult stage of sequences.
  • Strategies: A player could visually memorize the placement of the letters, try saying them outloud to gneerate an audible pattern, or create a pneumonic device.
